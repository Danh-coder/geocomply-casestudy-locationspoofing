# Design (1-page)

## 1) Ingestion and Scope

I modeled Android SDK telemetry as event-level rows with per-device temporal order. The pipeline ingests `train.csv` (labeled) and `test.csv` (labeled in current protocol) generated by `generate_data.py`.

- Train size: ~18k rows
- Test size: 5k rows
- Positive class: spoofed events (~20%)

Synthetic spoofing taxonomy includes:
- Teleportation (impossible movement)
- IP/Geo mismatch
- Mock provider usage
- Time drift manipulation
- Sensor inconsistency (high claimed precision with weak signals)

## 2) Features

Core location/context fields:
- `latitude`, `longitude`, `timestamp_utc`
- `reported_accuracy_m`, `reported_speed_mps`, `vertical_accuracy_m`
- `is_mock_location`, `location_permission_level`
- `ip_country`, `geo_country`, `vpn_active`, `proxy_active`
- `gnss_satellites_used`, `cell_signal_dbm`, `wifi_ap_count`, `wifi_rssi_dbm`
- `time_since_boot_s`, `clock_drift_seconds`

Derived features (rules + ML):
- `delta_t_s`, `distance_prev_m`, `implied_speed_mps`
- `speed_gap_mps`
- `geo_ip_mismatch`
- `clock_drift_abs_s`
- `accuracy_satellite_ratio`
- `radio_quality_score`

## 3) Rules Baseline

Rules are transparent and weighted:
- Impossible speed
- Mock location bit set
- Geo/IP mismatch with VPN/proxy
- Sensor conflict (high precision + low satellites)
- Large clock drift
- Speed inconsistency

Rules output:
- `spoof_score_rules` in [0,1]
- `spoof_flag_rules` at threshold 0.30

## 4) ML Model

A compact binary classifier uses Logistic Regression with:
- Standardized numeric features
- One-hot categorical features
- Binary pass-through features

Output:
- `spoof_score_ml` (probability)
- `spoof_flag_ml` (threshold from PR analysis)

## 5) AI/LLM Augmentation

`ai_helper.py` provides Gemini-backed helper functions for:
- Rule proposer (`propose_rules`)
- Weak labeler from text notes (`weak_label_batch`)
- Event explainer (`explain_events_batch` / `explain_event`)

`weak_label_from_note` remains local deterministic logic as a fallback/sanity path.

Batching policy:
- AI helper runtime calls are executed in chunks of 1,000 rows to reduce per-row API overhead.
- This is used for weak labeling and explanation generation in `model_train_eval.py`.

## 6) Decision Logic

Final score uses hybrid weighting:
- `hybrid_score = 0.75 * ml_score + 0.25 * rules_score`

Operating threshold is chosen from validation PR curve with precision-first target (0.88).

Final output for test:
- `results.json` records `{event_id, spoof_score, spoof_flag, explanation}`

## 7) Results Summary

Hybrid outperforms rules-only and slightly improves F1 over ML-only while preserving high precision. Detailed metrics and error analysis are in `EVAL_REPORT.md`.
